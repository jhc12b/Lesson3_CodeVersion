Highlight #1 Version Control System

A version control system (VCS) is a repository file that is usually used by team members that check the files out, make changes to them, and then check them back in. A VCS tracks theses changes according to who changed the file, when it was changed, what was modified in the file, and merges changes the same file.

Highlight #2 Git

Git is, according to their website, a free and open source distributed VCS designed to handle any kind of project with speed and effciency and is very easy to learn. Git is convenient because every user has a complete copy of the repository data stored locally on their machine, one can work offline, its processes run much faster, and git will prevent unintentional overwriting of a file on the shared repository.

Highlight #3 Repository

A repository is a data structure that stores metadata files and directory structure. Certain version control systems in use are distributed (remote) or centralized (local) , the whole set of information in the repository may be duplicated on every user's system.
